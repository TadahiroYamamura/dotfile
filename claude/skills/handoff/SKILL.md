# Handoff ドキュメント生成

プロジェクトの知見を引き継ぐための HANDOFF.md を生成し、`.agent/HANDOFF.md` に保存する。

**理由**: MEMORY.md はコンテキスト上限に近づくとトランケートされる。また、Claude Code 固有のメモリ構造は他のAIエージェントや別PC環境から参照できない。`.agent` に置くことでエージェント間・環境間で知見を共有できる。上書きでも追記でもなくアーカイブ方式にしたのは、過去のスナップショットを時系列で追えるようにするため。

## 手順

### 1. 情報収集

以下を並行して読み込む。

**メモリファイル**

カレントディレクトリのパスからプロジェクトメモリディレクトリを特定する。

パスのエンコード規則（例）:
- Windows: `C:\Users\foo\work\proj` → `C--Users-foo-work-proj`（`:` → `-`、`\` → `-`）
- macOS/Linux: `/home/foo/work/proj` → `-home-foo-work-proj`（`/` → `-`）

メモリディレクトリ: `~/.claude/projects/<encoded-path>/memory/`

`MEMORY.md` が存在すれば読み込み、インデックスに記載された各メモリファイルも読む。存在しない場合はスキップ。

**除外対象ファイル（重複チェック用）**

以下を読み込んでおき、内容が重複する知見はHANDOFF.mdに含めない:
- グローバル CLAUDE.md: `~/.claude/CLAUDE.md`
- プロジェクト CLAUDE.md: `.claude/CLAUDE.md`（存在する場合）
- AGENTS.md: `.agent/AGENTS.md`（存在する場合）

**現在のセッションの知見**

会話履歴を振り返り、以下を抽出する:
- 下した設計・技術的判断
- その判断に至った理由・背景
- 試してうまくいったこと、うまくいかなかったこと
- 次回以降に役立つ気づき

### 2. フィルタリング

収集した内容のうち、CLAUDE.md や AGENTS.md にすでに記載されているものは除外する。

HANDOFF.md に含める情報:
- **設計判断とその理由** — なぜその構成・技術・アプローチを選んだか。「何をしたか」は git log で分かるが「なぜか」は記録しないと失われる
- **セッションで得た知見** — 試行錯誤の結果、気づいたこと、有効だったパターン

### 3. HANDOFF.md の生成

プロジェクトルートに `HANDOFF.md` として出力する。

```markdown
# Handoff: [プロジェクト名またはディレクトリ名]
**生成日**: YYYY-MM-DD

## 設計判断と理由

### [判断のタイトル]
**判断**: [何を決めたか]
**理由**: [なぜそう決めたか]
**背景・制約**: [判断を左右した状況や制約があれば]

（複数ある場合は繰り返す）

## セッションで得た知見

- **[知見のタイトル]**: [内容。他のエージェントや別PCで読んでも意味が通じるよう自己完結した記述にする]

（メモリファイルに有益な内容があれば統合して記載する）
```

ファイルを出力したら、`.agent/HANDOFF.md` が存在する場合は diff を取って前回との差分をユーザーに見せる:

```bash
diff .agent/HANDOFF.md HANDOFF.md
```

差分のみを表示し、「前回からの差分です。内容を確認・編集してください。問題なければ保存する旨をお知らせください。」と伝える。

`.agent/HANDOFF.md` が存在しない場合（初回）は全文を表示する。

### 4. ユーザー確認待ち

ユーザーが確認・承認するまで待機する。ユーザーが内容を編集した場合はその編集内容を尊重する。

### 5. 保存

ユーザーの承認後、以下の順で処理する:

1. `.agent/handoff/` ディレクトリが存在しなければ作成する
2. `.agent/HANDOFF.md` が既に存在する場合:
   - そのファイルの最終変更日時を取得する
   - アーカイブ先 `.agent/handoff/HANDOFF-YYYYMMDD.md` が既に存在する場合は、`HANDOFF-YYYYMMDD-2.md`、`HANDOFF-YYYYMMDD-3.md` のように連番サフィックスを付けて重複を回避する
   - `.agent/handoff/HANDOFF-YYYYMMDD.md` としてリネーム（YYYYMMDDは最終変更日）
3. プロジェクトルートの `HANDOFF.md` を `.agent/HANDOFF.md` に移動する
4. 「`.agent/HANDOFF.md` に保存しました。」と報告する

## 注意事項

- HANDOFF.md は複数の AI エージェントや別環境から読まれることを想定する。プロジェクト固有の文脈を含め、自己完結した記述にする
- 箇条書きより文章・構造化された記述を優先する（構造的な知識を伝えやすいため）
- 「なぜ」を重視する。実装の詳細より判断の背景が重要
- セクションに書くことがなければそのセクションごと省略してよい
