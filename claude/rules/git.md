# Git ノウハウ

## コミット前の確認

- コミットする前に**必ずユーザーの承認を得る**

## ブランチ管理

- ブランチを切る前に developブランチで `git pull origin develop` を実行して最新化する
- サブモジュールがある場合は `git submodule update --remote` で最新化する
- プッシュはユーザーが明示的に指示した時のみ実行する
- 作業中にマージ済みのローカルブランチを見つけた場合、削除を提案する

## コミットメッセージ

フォーマット:

```
<type>(<scope>): <summary>

## 意図
<なぜこの変更をしたか>

## 設計上の考慮点
<実装にあたって検討したこと・選択の理由>

## 懸念事項
<既知の制限・将来的なリスク等。なければ省略>
```

| type | 用途 |
|---|---|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `docs` | ドキュメントのみの変更 |
| `refactor` | 動作を変えないコードの整理 |
| `test` | テストの追加・修正 |
| `chore` | ビルド・設定・依存関係など雑務 |

- `summary` は命令形・体言止めで50文字以内
- `## 意図` は必須
- `## 設計上の考慮点` は実装の選択が非自明な場合に記載
- `## 懸念事項` は既知の制限やリスクがある場合のみ記載（なければ省略）

## stash から特定ファイルを選択的に取り出す

stash から特定ファイルだけを取り出したい場合は、`git stash pop` で全て戻してから不要なものを再度 `git stash` する。

追跡済みファイルは `git checkout stash@{0} -- <file>` で取り出せるが、untracked ファイルを `stash@{0}^3` で取り出す方法は公式ドキュメント外の実装依存のため使わない。
